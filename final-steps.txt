WHAT TO CHECK IF SOMETHING FAILS
================================================================================

TESTS FAILING (pytest)
  - Run `pytest tests/ -v` to see which test fails.
  - Most likely cause: a file was accidentally modified. Check git diff.
  - If test_environment.py fails: the rewards or grading behaviour may have drifted.
  - If test_graders.py fails: check server/grading.py was not overwritten.

IMPORT ERRORS (ModuleNotFoundError)
  - Make sure venv is activated and `pip install -r requirements.txt` was run.
  - Check Python version: `python --version` — must be 3.9 or higher.

/reset NOT RETURNING 200
  - Make sure the server is running: `python -m uvicorn server.app:app --port 7860`
  - Check for port conflict: `netstat -ano | findstr 7860` (Windows)
    Kill or use a different port: `--port 7861` and set `ENV_URL=http://localhost:7861`
  - Check server logs — an import error at startup will appear in stderr.

DOCKER BUILD ERRORS
  - "Cannot connect to Docker daemon" → Docker Desktop is not running. Open it.
  - pip install fails in Dockerfile → check requirements.txt versions.
    Try adding `--no-cache-dir` (already present) and check network access.
  - "COPY failed" → make sure .dockerignore is not accidentally excluding needed files.

SPACE BUILD FAILURES (HF Logs)
  - Check the Logs tab on the Space page.
  - "pip install failed" → same as Docker build errors above.
  - "port 7860 not listening" → the CMD in Dockerfile must bind to 0.0.0.0:7860.
    Current CMD: ["uvicorn","server.app:app","--host","0.0.0.0","--port","7860"] ✓
  - Build timeout → Space free tier has limited CPU. Try again; if persistent,
    check if a dependency is being compiled from source (add a prebuilt wheel).

openenv VALIDATE FAILURE
  - Run: `python -m openenv.cli validate`
  - If "field required" error → a required field is missing from openenv.yaml.
    Compare against the current openenv.yaml in the repo.
  - If "unknown field" error → remove the offending field from openenv.yaml.
  - The current openenv.yaml passes with: [OK] cloud-alert-triage: Ready for multi-mode deployment

INFERENCE AUTH FAILURE (401 / AuthenticationError)
  - Check OPENAI_API_KEY is exported and non-empty: `echo $OPENAI_API_KEY`
  - If using HF Inference API, make sure the model is available on free tier.
  - Check API_BASE_URL ends with /v1 (no trailing slash issues).

BAD LOGGING FORMAT
  - inference.py must emit exactly:
      [START] {"task":"...","env":"cloud-alert-triage","model":"..."}
      [STEP] {"step":N,"action":{...},"reward":F,"done":B,"error":null}
      [END] {"success":B,"steps":N,"score":F,"rewards":[...]}
  - Check that inference.py has not been replaced. The current version uses
    `from openai import OpenAI` and the log_start/log_step/log_end functions.
  - Run: `python -c "import inference; print('OK')"` to check for import errors.
